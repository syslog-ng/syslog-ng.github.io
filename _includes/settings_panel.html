{% comment %} Settings panel popover toggled by the masthead `#settings-button`. Persistence: - Dark mode -> reuses the existing `skin-state` cookie owned by skins.html. - Prefill -> cookie `settings-prefill-selection` ('true' / 'false', default 'true'). - ESC mode -> cookie `settings-esc-behavior` ('close' | 'clear-then-close' | 'close-and-clear', default 'close'). - Hotkey -> cookie `settings-hotkey-search` (combo string, e.g. 'Ctrl+Shift+F'; empty string disables the shortcut). - Tooltip lag -> cookie `settings-tooltip-delay` (ms, default '100'). Consumers: - navigation.js reads these cookies on demand at event time. - The dark-mode checkbox here just forwards the click to `#skin-button` so we do not duplicate skin-state logic. {% endcomment %}
